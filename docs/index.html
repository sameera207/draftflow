How it works
The round-trip workflow.
01

Type /df in Claude Code

The /df slash command writes your content to a shared bridge file and opens it in Draftflow automatically.

02

Edit with full autocomplete

Your file opens in the editor. Type # to fuzzy-match skills, browse related files, and refine your draft.

03

Click "Send back"

Draftflow writes to ~/.claude/editor-bridge/response.md and Claude Code picks it up immediately.

04

Or use standalone

Open any .md file, write a prompt, and copy it to your clipboard ready to paste into any Claude session.

Quick start
Up and running in seconds.
Terminal
$ git clone https://github.com/sameera207/draftflow.git
$ cd draftflow
$ npm install
$ npm start
✓ Draftflow is running
# Install the /df Claude Code command
$ open Draftflow → Settings → Install /df command
FAQ
Common questions.
No API key required. Draftflow is a local markdown editor. It doesn't call any AI APIs — it integrates with Claude Code running in your terminal, which handles its own authentication.
As you type, pressing # opens a fuzzy search over your installed Claude Code skills (markdown files in your configured scan paths). Select one to insert a reference to it in your draft.
Draftflow targets macOS at initial launch. It's built on Electron so it may work on Linux and Windows, but these platforms are not officially supported or tested yet.
The "Send to Claude" button copies your current markdown to the clipboard as a formatted prompt. You then paste it into a Claude Code session in your terminal. No browser integration, no magic — just clipboard.
Yes. Draftflow is fully open source under the MIT license. Fork it, extend it, and make it yours. Contributions welcome.
